## LAYER 11 — DEPLOY CONTEXT (per-task overlay)

You are scaffolding deploy artefacts for Day 11. The 11 Lambdas + state
machine + dashboard + alarms already exist (Days 1-10 + pretraffic-hook).
You're converting manual `aws ... update-function-configuration` style
provisioning to SAM + GitLab CI + OIDC.

Authoring rules:

1. ALWAYS use the TWO-STACK SPLIT:
   - `infra/template-foundation.yaml` — DDB, Kinesis, SQS, SNS only
   - `infra/template-app.yaml` — Lambdas, SFN, dashboard, alarms only
   App stack imports foundation outputs via `!ImportValue accelya-foundation-<learner>-<OutputName>`.
   NEVER duplicate stateful resources in app stack.

2. EVERY stateful resource in foundation MUST have BOTH:
       DeletionPolicy: Retain
       UpdateReplacePolicy: Retain
   Both required — only one means an in-place update can still destroy the resource.

3. EVERY Lambda in app stack MUST have:
       AutoPublishAlias: live
       DeploymentPreference:
         Type: Linear10PercentEvery1Minute
         Hooks:
           PreTraffic: !GetAtt PreTrafficHookFunction.Arn
         Alarms:
           - !Ref IteratorAgeAlarm    # at minimum, this one
           - !Ref BedrockThrottleAlarm  # plus relevant ones per function
   ONLY exception: the PreTraffic-hook Lambda itself (self-reference illegal).

4. EVERY Lambda MUST have an EXPLICIT log group (NOT auto-created by runtime):
       <FnName>LogGroup:
         Type: AWS::Logs::LogGroup
         DeletionPolicy: Retain
         UpdateReplacePolicy: Retain
         Properties:
           LogGroupName: !Sub /aws/lambda/<fn-physical-name>
           RetentionInDays: 14
   And the Function:
       <FnName>Function:
         DependsOn: <FnName>LogGroup
         Properties:
           LoggingConfig:
             LogGroup: !Ref <FnName>LogGroup

5. OIDC trust policy MUST use:
       Action: sts:AssumeRoleWithWebIdentity
       Condition:
         StringEquals:
           gitlab.com:aud: sts.amazonaws.com  # NEVER any other value
         StringEquals (protected main) OR StringLike (feature branches) on:
           gitlab.com:sub: project_path:<group>/<project>:ref_type:branch:ref:<branch-or-glob>
   NEVER use sub: '*' or sub: 'project_path:*' or aud: '*'.

6. `.gitlab-ci.yml` MUST have 4 stages:
       lint → test → deploy-sandbox → integration-test
   Lint runs 4 scanners in PARALLEL: bandit + semgrep + cfn-lint + cfn-nag.
   NOT tflint — that's Terraform-only; wrong tool for SAM/CFN.
   `deploy-sandbox` job MUST have `environment:on_stop: cleanup-sandbox`
   AND `auto_stop_in: 4 hours` as the backstop.

7. `id_tokens:` block in EVERY job that hits AWS:
       id_tokens:
         GITLAB_OIDC_TOKEN:
           aud: sts.amazonaws.com
   NEVER set AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_SESSION_TOKEN
   in CI variables. Period. The `preflight-lambda-cap.sh` job grep-checks
   `glab variable list` to enforce this.

8. `sam deploy` invocation MUST include `--capabilities CAPABILITY_IAM`
   (or CAPABILITY_NAMED_IAM if your role uses RoleName). Without it,
   sam deploy fails with InsufficientCapabilities.

9. The PreTraffic hook handler (src/pretraffic_hook/handler.py):
   - Reads `DeploymentId` and `LifecycleEventHookExecutionId` from event
   - Invokes the target Lambda synchronously with a synthetic event
   - Returns 'Succeeded' iff status_code==200 AND no FunctionError
   - Calls `codedeploy.put_lifecycle_event_hook_execution_status(...)`
   - Wraps ALL exceptions — never crash the hook itself

10. The migration gate (Lab 1 Step 0 — TRAINER ONLY):
    - `bash scripts/preflight-lambda-cap.sh <learner> --delete-manual`
    - Snapshots all `accelya-*-<learner>` Lambdas to reviews/m11-pre-migration/
    - Deletes each manually-created Lambda
    - Verifies count==0 before sam deploy
    - LEARNERS SKIP this step because their daily-reset sandbox has nothing
      to migrate from.

[LAYER 10] TASK
Author the four streaming Lambdas + state machine with these contracts:

(A) src/producer/handler.py
    Event:  { "events": [<envelope>, ...] }  (≤500 per call)
    Return: { submitted_count, failed_count, failed_records[] }
    Behavior:
      - Pre-validate each envelope (raise ValidationError pre-submit)
      - Bulk PutRecords; PartitionKey = f"{airline_code}:{booking_date}:{bucket_hash(pnr)%8}"
      - On per-record PutRecords failure (ProvisionedThroughputExceededException):
          backoff+jitter, retry the failed records only, cap 5 retries
      - EMF: records_in, records_failed, retry_count (tags: stream_name, airline_code)

(B) src/consumer/handler.py — THE THIN CONSUMER
    Event:  ESM { "Records": [{ "kinesis": {partitionKey, sequenceNumber, data},
                                 "eventID": "shardId-...:sequenceNumber" }, ...] }
    Return: { "batchItemFailures": [{"itemIdentifier": "<oldest_failed_seq>"}] }
    Four stages (in order, per record):
      1. Claim stream_seen_keys PK = f"stream#{STREAM}#{shard_id}#{seq}"
         (ConditionalPut, attribute_not_exists)
         Duplicate → silent skip
      2. base64 decode data; validate envelope; ValidationError → quarantine + ACK
      3. sfn.start_execution(stateMachineArn=SM_ARN, name=f"enrich-{event_id}", input={...})
         ExecutionAlreadyExists → idempotent skip
      4. UpdateItem stream_seen_keys.pk: status = COMPLETED
    Failure handling:
      - ValidationError    → quarantine/<learner>/stream_invalid/<reason>/  (ACK, not failure)
      - TransientError     → append seq to failures
      - Report ONLY the oldest failure per shard
    NO inline Bedrock. NO inline S3 enriched writes. NO non-deterministic exec names.

(C) src/bedrock_enrich/handler.py
    Step Functions Task invocation. Sampled enrichment.
    Steps:
      1. should_enrich(event_id, SAMPLE_RATE): MD5 % SAMPLE_RATE == 0
         If false → return {**event, enrichment: None, sampled: False}
      2. Pre-flight CW budget check: AWS/Bedrock Invocations + Tokens for today
         vs BEDROCK_MAX_INVOCATIONS / BEDROCK_MAX_TOKENS
         If at cap → return {**event, enrichment: None, sampled: True,
                              budget_guard_skipped: True}
      3. boto3.client('bedrock-runtime').invoke_model with apac Sonnet profile;
         tight prompt (~150 in / ~100 out tokens); max_tokens=256
         Exceptions:
           ThrottlingException / ServiceUnavailableException → raise BedrockThrottleError
           AccessDeniedException → raise ConfigError
      4. EMF: bedrock_invoked / bedrock_skipped / bedrock_throttled /
              budget_guard_skipped / input_tokens / output_tokens
              (Dimensions: ModelId, Function)
    Return: {**event, enrichment, sampled, enriched_at}

(D) src/persist_enriched/handler.py
    Step Functions Task. Writes enriched record to S3.
    Path: enriched/<learner>/ingest_date=YYYY-MM-DD/source_system=GDS/events-<event_id>.jsonl.gz
    Gzip-compress JSON line; ContentEncoding=gzip.
    Return: { persisted_key, outcome: "persisted" }

(E) infra/inference-pipeline.asl.json
    StartAt: BedrockEnrich
    BedrockEnrich Task → accelya-bedrock-enrich-<learner>
      Retry: BedrockThrottleError → 2s × 3 backoff 2.0
      Retry: Lambda.* exceptions → 2s × 3 backoff 2.0
      Catch: States.ALL → ResultPath $.error → Next Quarantine
      Next: Persist
    Persist Task → accelya-persist-enriched-<learner>
      Retry: Lambda.* exceptions → 2s × 3 backoff 2.0
      End: true
    Quarantine Pass state → End: true

(F) src/common/exceptions.py
    Add: class BedrockThrottleError(PipelineError): quarantine_reason = "bedrock_throttle"

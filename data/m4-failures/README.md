# M4 Failures — Synthetic Broken Files

> **5 files used by Day 4 Lab 2.** Each file demonstrates ONE of the 6 failure modes from Concept 4.
>
> **Important — these files are intentionally broken.** Do NOT try to "fix" them. The whole point of Lab 2 is to walk each one and trace where it would be caught in the pipeline.

---

## What's in this folder

| File | Category | What's deliberately wrong | Catch point |
|---|---|---|---|
| `01-schema-drift.json` | Schema drift | Row 47 has `fare_currency` instead of `currency`; row 89 has `fare_amount` as string instead of number | `schema_check()` → `quarantine/schema_drift/` |
| `02-partial.json` | Partial write | File truncated mid-row 31; JSON is intentionally invalid | `manifest_row_count_check()` → `quarantine/partial/` |
| `03-duplicate.json` | Duplicate ingest | Byte-identical copy of `01-schema-drift.json` | `claim('file#' + file_sha256)` → DynamoDB no-op (NOT quarantine) |
| `04-late-event.json` | Late event | All 50 rows have `issue_date` from 95 days ago | `event_date_check()` → `validated/` with `is_late=true` (NOT quarantine) |
| `05-malformed.json` | Malformed row | Row 14 has unescaped quote; row 67 has trailing comma; JSON is intentionally invalid | `json.loads()` raises → `quarantine/malformed/` |

**Sixth category — source outage — is design-only; no file.** Lab 2 step 5 asks you to write the design section for it (EventBridge scheduled checker + SNS paging).

## Why these files live in the repo (and not in S3)

The original design called for these to be seeded under `s3://accelya-airline-data-ap-south-1/raw/synthetic/m4-failures/`. We moved them into the repo because:

1. **Learners may not have read access to that S3 prefix** — depends on per-learner IAM scoping which varies by cohort.
2. **Day 4 is design-only.** Lab 2 inspects these files locally — no need to round-trip through S3.
3. **Reproducible.** The repo is the source of truth; everyone has the same files at the same commit.

When you move to Day 5 (real implementation), the files get uploaded to S3 by the trainer's seed script — but the Day 4 lab itself reads from local copies.

## How to inspect each file

```bash
cd accelya-agentic-aws

# Inspect a valid file (01)
head -30 data/m4-failures/01-schema-drift.json
python3 -c "import json; r=json.load(open('data/m4-failures/01-schema-drift.json')); print(f'row 47: {r[47]}')"

# Inspect a broken file (02 — won't parse as JSON)
head -50 data/m4-failures/02-partial.json
python3 -c "import json; json.load(open('data/m4-failures/02-partial.json'))" 2>&1 | head -3

# Diff 03 vs 01 (should be identical)
diff data/m4-failures/01-schema-drift.json data/m4-failures/03-duplicate.json
# Output: (no difference)

# Check 04's date
python3 -c "import json; print(json.load(open('data/m4-failures/04-late-event.json'))[0]['issue_date'])"

# 05 — find the malformed rows
grep -n '0014"' data/m4-failures/05-malformed.json   # row 14 — unescaped quote
grep -n ',,' data/m4-failures/05-malformed.json      # row 67 — trailing comma
```
